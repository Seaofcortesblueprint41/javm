# JAVM v0.1.6

## 概览

- 发布范围：`v0.1.5..HEAD`
- 修复 Windows Release 工作流在下载上游依赖时可能触发 GitHub API 匿名限流的问题。
- 发布流程改为优先使用仓库内置的 GitHub token 访问 Releases API，提升自动构建稳定性。

## 新增

- 无面向最终用户的新功能。

## 优化

- Release 工作流在请求 `nilaoda/N_m3u8DL-RE` 和 `BtbN/FFmpeg-Builds` 的 GitHub Releases API 时，改为附带认证头与 API 版本头，减少共享 runner IP 导致的限流失败。
- 当上游 GitHub API 仍返回限流时，工作流会输出更明确的错误说明，方便快速定位为认证或配额问题，而不是误判为资源缺失。

## 修复

- 修复正式版构建过程中 `Invoke-RestMethod` 访问 GitHub API 可能直接因 `API rate limit exceeded` 失败，导致安装包构建中断的问题。

## 兼容性与迁移

- 无特殊操作要求。

## 已知问题

- 若 GitHub Actions 的默认 token 权限被仓库策略额外收紧，仍可能导致上游 Release API 请求失败，此时需要检查仓库 Actions 权限配置。
