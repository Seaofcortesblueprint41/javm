# JAVM v0.1.5-beta.5

## 概览

- 发布范围：`v0.1.5-beta.4..HEAD`
- 本次测试版继续聚焦跨平台下载链路与 GitHub Actions 发布流程验证。
- 重点改进了下载器在 Windows、macOS、Linux 上的可执行文件路径解析，并补充了更稳定的 macOS FFmpeg 下载来源。

## 新增

- GitHub Actions 发布流程新增 FFmpeg 下载步骤，按平台和架构准备打包所需的二进制资源。
- macOS 发布任务改为通过脚本化重定向地址下载对应架构的 FFmpeg，便于后续持续集成自动化维护。

## 优化

- 下载器路径解析逻辑已适配跨平台命名差异，减少 Windows、macOS、Linux 下找不到可执行文件的情况。
- 默认下载目录改为通过 Tauri 路径 API 获取，避免依赖硬编码路径带来的环境差异问题。
- 多处 HTTP 请求统一使用更完整的 User-Agent，降低部分站点因请求头过于简化导致的兼容性问题。

## 修复

- 修复跨平台下载场景下下载器可执行文件命名不一致导致的资源定位失败问题。
- 修复远程图片下载请求头过于简单时可能触发目标站点拦截的问题。
- 补充 Linux 构建依赖，降低 CI 打包阶段因缺少系统组件而失败的概率。

## 兼容性与迁移

- 无特殊操作要求。

## 已知问题

- 本版本仍为测试预发布版本，主要用于验证跨平台下载资源准备与 CI Release 打包流程。
- macOS Intel 构建当前依赖第三方 FFmpeg 构建源的持续维护，后续仍需持续观察上游可用性。
