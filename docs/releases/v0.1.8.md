# JAVM v0.1.8

## 概览

- 发布范围：全量初始发布（git 历史已重置）
- FFmpeg 构建来源切换为 ddmoyu/ffmepg 预编译裁剪版，大幅缩减二进制体积并消除网络协议等无关能力。
- 修复 Windows 平台下使用 FFmpeg 截图时终端窗口反复闪烁的问题。
- 移除对 ffprobe 的依赖，视频时长获取与诊断统一使用裁剪版 ffmpeg 完成。

## 新增

- 无面向最终用户的新功能。

## 优化

- GitHub Actions Release 工作流中 FFmpeg 下载源从 BtbN/FFmpeg-Builds（全量 GPL 构建）和 evermeet（macOS）切换为 ddmoyu/ffmepg 统一预编译源，六个平台使用同一套下载逻辑，维护更简单。
- 裁剪版 FFmpeg 仅保留本地视频输入、抽帧截图所需能力，关闭网络协议、硬件加速、音频转码和 ffprobe/ffplay，安装包体积显著缩小。
- 所有 FFmpeg/ffprobe 子进程在 Windows 上设置 `CREATE_NO_WINDOW` 标志，截图和获取时长时不再弹出控制台窗口。
- `get_video_duration()` 改为解析 `ffmpeg -i` 输出中的 `Duration: HH:MM:SS.ss` 行，不再依赖 ffprobe。
- `diagnose_video_at_time()` 改为用 ffmpeg 读取目标时间点 1 帧到 null 输出，通过 stderr 判断位置可读性，不再依赖 ffprobe。

## 修复

- 修复 Windows 平台下每次调用 FFmpeg 截图或获取视频时长时，都会短暂弹出黑色终端窗口的问题。

## 兼容性与迁移

- 本版本起不再需要 ffprobe，仅需 ffmpeg 可执行文件。
- GitHub Actions 构建要求 `gh` CLI 可用（GitHub Hosted Runner 默认已安装）。
