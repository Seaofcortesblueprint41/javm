# JAVM v0.1.10

## 概览

- 发布范围：`v0.1.9..HEAD`
- 新增应用内更新能力，支持在应用内检查新版本、查看更新说明并安装更新。
- 重构媒体刮削元数据与资源命名，补齐题材分类、封面、预览图等字段，提升媒体整理与展示一致性。
- 新增“仅删除刮削数据”模式，允许保留原始视频文件，仅清理 NFO、封面图等派生资源。
- 扩展字幕识别规则并修复多轮发布中的构建与更新链路问题，为正式版提供更稳定的安装与升级体验。

## 新增

- **应用内更新**：应用启动后可自动检查新版本，也可在设置页手动触发检查，并在更新弹窗中查看版本说明后再决定是否升级。
- **仅删除刮削数据**：删除视频时可选择只删除 NFO、封面图、预览图等刮削产物，保留本地视频文件，适合需要重新刮削但不想动原片的场景。
- **更完整的刮削元数据**：新增 `genres` 等元数据字段，并统一封面、缩略图、背景图等资源模型，便于后续展示、导出与落盘。
- **旧数据库结构识别与重建**：应用启动时会检测旧版本地数据库 schema，必要时自动重建，减少跨版本升级后因旧结构不兼容导致的初始化失败。

## 优化

- **媒体资源命名统一**：将原有 `screenshots` 等概念统一为更清晰的资源命名，并规范预览图目录到 `extrafanart`，减少前后端字段语义不一致。
- **字幕匹配更宽松**：新增 `sbv`、`dfxp`、`ttml`、`scc`、`usf` 等字幕扩展名支持，并允许更多命名分隔符参与匹配，提升本地字幕识别成功率。
- **视频信息处理更稳**：编辑视频时增加标题合法性校验，保存后会同步刷新列表与当前选中项，减少修改后界面仍显示旧数据的情况。
- **媒体元数据提取增强**：补充 FFmpeg 兜底逻辑，并优化下载任务文件组织结构，方便资源整理与后续刮削落盘。
- **发布与升级链路稳定性提升**：围绕 updater 公钥注入、release workflow 配置和构建类型问题做了多轮修正，正式版构建与后续在线升级链路更可靠。

## 修复

- 修复部分字幕文件因扩展名较少或命名分隔符不一致，导致无法正确识别和关联的问题。
- 修复删除视频时只能连同原始视频一起清理的问题，现在可以按需只删除刮削数据并保留媒体文件。
- 修复数据刮削弹窗中预览图类型处理不稳定导致的构建失败，避免发布构建被 TypeScript 类型错误阻塞。
- 修复旧版数据库直接升级时可能沿用过期 schema，导致后续初始化失败或运行异常的问题。
- 修复发布流程中 updater 公钥读取与注入不稳定造成的多平台构建失败问题，降低正式版发版中断风险。

## 兼容性与迁移

- 升级到本版本时，如检测到旧版本地数据库 schema，应用可能会自动删除旧数据库并重建；如你仍需保留旧数据，请先自行备份。
- 应用内更新依赖 GitHub Release 资产、更新清单以及仓库中的 updater 签名配置；正式发布后才能获得完整在线升级体验。
- 无其他特殊操作要求。
